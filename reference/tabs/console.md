# Console Tab
**Runtime Atlas v1.2.0**

---

## Purpose

Captures Unity log messages that are prefixed with a Runtime Atlas tag. Functions as a scoped console view showing only messages generated by Runtime Atlas itself or by user code that adopts the prefix convention.

---

## Captured Prefixes

The Console tab captures messages with any of these prefixes:

- `[RA]`
- `[RuntimeAtlas]`
- `[Runtime Atlas]`

Messages without these prefixes are not captured and remain in the standard Unity Console only.

---

## Log Entry Fields

| Field | Description |
|-------|-------------|
| Type | Log / Warning / Error |
| Message | Log message text |
| Timestamp | Frame index at capture time |
| Stack trace | Expandable stack trace (click the row to expand) |

---

## Controls

| Control | Action |
|---------|--------|
| Log / Warning / Error toggles | Filter by message type |
| Search field | Filter by message text |
| **Clear** | Remove all captured messages |

---

## Ring Buffer

The console maintains a fixed-size ring buffer (default: 500 entries). When the buffer is full, oldest entries are overwritten. The buffer size is configurable in settings.

---

## Using the Prefix in Your Code

To have your own log messages appear in the Runtime Atlas Console tab:

```csharp
Debug.Log("[RA] Your message here");
Debug.LogWarning("[RuntimeAtlas] Your warning here");
```

---

## Limitations

The Console tab captures messages generated at runtime (after `Application.logMessageReceived` is registered). Messages generated before the Runtime Atlas window is opened are not captured.
